引言

以前都是自己写着笔记，想着建一个博客，但是又有点懒，后来吃同学安利，了解到了hexo博客，于是就想着整个hexo博客吧，期间踩了不少坑，记录一下建站过程。

Hexo

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

Hexo官网的文档已经有好几年的历史了，上面的文档对于新手来说比较难读懂。

阶段一：使用Hexo搭建个人博客

环境准备

搭建个人博客需要提前整好这两个环境，后面才能搞起来，如果没有学习过，建议学习一下后安装环境。

  1. Git，Git 是目前世界上最先进的分布式版本控制系统，可以有效、高速的处理从很小到非常大的项目版本管理。我们这次个人博客的搭建就是基于 github pages。可以看廖雪峰的网站，写得挺详细的。
  2. Node.js环境， Chrome V8 引擎的 JavaScript 运行环境。Node.js 的包管理器 npm, 是全球最大的开源库生态系统。我们所使用到的 hexo，就是基于 nodejs 编写的，需要使用到 nodejs 的包管理器 npm一个基于。可以看一下菜鸟教程。

1安装hexo

所有必备的应用程序安装完成后，即可使用 npm 安装 Hexo。

接着使用管理员身份打开cmd命令行工具，不然下面这个例子可能安装不了hexo，可能会报没有权限新建文件夹。

接着通过cd命令进入到自己要安装博客的位置，初始化一下自己的hexo

新建完成后，指定文件夹目录下有：

  • node_modules: 依赖包
  • public：存放生成的页面
  • scaffolds：生成文章的一些模板
  • source：用来存放你的文章
  • themes：主题
  • _config.yml: 博客的配置文件

模版（Scaffold）

/souce/scaffolds里默认有这三个模板，这三个模板可以打开修改的，这里只讲怎么用模板

  • draft.md是草稿的模板，新建后的草稿会放在/source/_drafts里
  • post.md是新文章的模板，新建后的文章会放在/source/_posts里
  • page.md是页面的模板，新建后的页面会放在/source/里，和上面两个不同，这个不是新建文章的，而是新建一个页面的，可以通过配置把新建出来的页面加到页面的导航栏中。

Front-matter

Front-matter 是每一个markdown文件最上方以 --- 分隔的区域，用于指定个别文件的变量。

⭐非常重要，特别是categories信息，直接指定文章的分类！

参数 描述 默认值

title 标题 文章的文件名

excerpt Page excerpt in plain text. Use this plugin to format the text

只有文章支持分类和标签，您可以在 Front-matter 中设置。在其他系统中，分类和标签听起来很接近，但是在 Hexo 中两者有着明显的差别：分类具有顺序性和层次性，也就是说 Foo, Bar 不等于 Bar, Foo；而标签没有顺序和层次。
